Proof, not promises

I've rebuilt a live Bullhorn environment at the database layer

Not advice from a slide deck — hands-on work inside a working agency's Bullhorn Enterprise instance, while the team kept selling and recruiting around me. What I built there:

5,821 candidate records audited

Rebuilt the category taxonomy from scratch — 9 clean, recruiter-validated categories — and ran a structured bulk-tagging operation mapping the full database to it, so every search returns people the team can trust.

13 statuses consolidated into one working model

Moved 2,683 records out of ambiguity buckets and organized 3,601 dormant candidates into structured re-engagement lists instead of dead weight.

A 221-lead sales pipeline, built from the object model up

Defined what Lead, Contact, Company, and Job Order mean operationally, converted misfiled records, and stood up a full lifecycle with lead-source attribution — pipeline visibility where there was none.

Bulk operations through the Bullhorn REST API

Secured and structured API access for mass updates and automation beyond what the interface allows — including an automation that keeps eligible candidates visible after assignments end.

A 17-certification compliance framework

Expiry tracking across licenses, equipment, and safety certifications — stood up from nothing, maintained in the system, not a spreadsheet.

SOPs, training, and adoption

Written usage standards, role-based training, and a 379-record triage tracker — the unglamorous work that decides whether any cleanup lasts.
